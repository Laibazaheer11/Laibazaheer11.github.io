Debugging · Real Example

A bug I found in my own test script

Not a hypothetical — this is something I actually got wrong, caught, and fixed.

🐞 Array-Indexing Bug self-caught
// Before — treats a single-object response like an array pm.expect(data[0].id).to.eql(pm.globals.get("userid_A")); // GET /users/{id} returns ONE object, not an array — data[0] was undefined // After — fixed pm.expect(data.id).to.eql(pm.globals.get("userid_A"));

Caught while testing GoRest and Simple Books API single-resource endpoints — a good reminder to check response shape before writing assertions.
